Silver Focus 03

抽象クラスとインタフェースは「契約」を読む。

ここは暗記だけで押し切ると崩れます。何を実装しなければならないか、どのメソッドが自動的にpublicなのか、default/static/privateがどこに書けるかを整理します。

interface Printable {
  void print();

  default void log() {
    System.out.println("log");
  }
}

abstract class Report {
  abstract String title();
}

class DailyReport extends Report implements Printable {
  public void print() {
    System.out.println(title());
  }

  String title() {
    return "daily";
  }
}
abstract

抽象メソッドは中身がない約束

抽象クラスは状態や実装を持てますが、抽象メソッドを残したままnewできません。具象クラスが実装する必要があります。

interface

インタフェースは実装の契約

実装クラスは、抽象メソッドをすべて実装します。インタフェースのメソッドは基本的にpublicとして扱われます。

default

defaultメソッドは本体を持てる

Java 8以降、インタフェースにもdefaultメソッドを書けます。実装クラスで上書きしない場合はその実装を使えます。

static

staticメソッドは型名から呼ぶ

インタフェースのstaticメソッドは実装クラスへ継承される感覚で呼びません。どの型に属するかを見ます。

access

弱いアクセスにできない

publicな契約を、実装側でpackage-privateやprivateに狭めることはできません。コンパイルエラーの定番です。

multiple

複数実装の衝突を見る

複数インタフェースのdefaultが衝突したら、クラス側で明示的に解決が必要です。曖昧なままにはできません。

Tracea Note

「書ける場所」と「呼べる形」を分ける。

抽象クラスはフィールド、コンストラクタ、具象メソッドを持てます。インタフェースは契約が中心ですが、default/static/privateメソッドも扱います。似ているからこそ、何がどこに書けるかを表で持つと強いです。

試験では、見た目が正しそうなコードでもアクセス修飾子で落ちます。インタフェース由来のメソッドを実装するとき、publicを落としていないか。そこに赤線を引いて読みましょう。
