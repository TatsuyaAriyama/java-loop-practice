Silver Focus 01

StringとArrayListは、出力結果をずらす主役です。

Silverでは「知っているメソッド名」より、呼んだあとに元の値が変わるのか、戻り値だけが新しいのかを見ます。ここで目を止められるかどうかで、選択肢の消し方が変わります。

String text = "Java";
text.concat("Silver");
System.out.println(text);

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.remove(0);
System.out.println(list.get(0));
immutable

Stringは変更できない

concatやreplaceは、新しい文字列を返します。元の変数に代入していなければ、表示される値は変わりません。

index

文字もリストも0番から数える

charAt(0)は先頭、get(0)も先頭です。人間の「1番目」とJavaの0がずれます。

equals

文字列比較はequalsを見る

==は同じ参照かを見ます。内容が同じかを問う問題では、基本的にequalsが主役です。

ArrayList

ArrayListは要素数が変わる

add、removeで中身が動きます。削除後は後ろの要素が前へ詰まる。この一歩を雑に読むと出力を外します。

overload

removeは引数で意味が変わる

remove(0)は0番を消し、remove("0")は値としての"0"を探します。型を見ます。そこです。

pitfall

試験では「変わったつもり」を狙う

Stringは変わらない。ArrayListは変わる。まずこの対比を置いてから、1行ずつ値を追いましょう。

Tracea Note

戻り値を捨てていないか、変更が本体に効くのか。

Stringのメソッドは、結果を返しても元のStringオブジェクトは変えません。つまり text.concat("Silver"); だけでは、textはまだ "Java" です。ここを見落とすと、正解は一気に遠ざかります。

一方でArrayListは中身を変えます。removeした瞬間、要素の位置は詰まります。問題文に複数のadd/removeが並んだら、頭の中だけで耐えず、0番、1番、2番と小さな表を作るのが勝ち筋です。
