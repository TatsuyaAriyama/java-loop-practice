String / Builder / List

文字列とリストは、「変わるか・変わらないか」で読む。

String、StringBuilder、ArrayListはどれも値を扱います。ただし、メソッドを呼んだあとに本体が変わるかどうかが違います。ここを見抜けると、出力結果の読み違いが一気に減ります。

String text = "Java";
text.concat("Silver");
System.out.println(text);

StringBuilder sb = new StringBuilder("Ja");
sb.append("va");
System.out.println(sb);

ArrayList<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.remove(0);
System.out.println(list.get(0));
immutable

Stringは変更できない

concatやreplaceは、新しい文字列を返します。元の変数に代入していなければ、表示される値は変わりません。

index

文字もリストも0番から数える

charAt(0)は先頭、get(0)も先頭です。人間の「1番目」とJavaの0がずれます。

equals

文字列比較はequalsを見る

==は同じ参照かを見ます。内容が同じかを問う問題では、基本的にequalsが主役です。

StringBuilder

StringBuilderは中身が変わる

append、insert、deleteは、そのStringBuilder自身を変えます。Stringとの違いを見ます。

ArrayList

ArrayListは要素数が変わる

add、removeで中身が動きます。削除後は後ろの要素が前へ詰まる。この一歩を雑に読むと出力を外します。

overload

removeは引数で意味が変わる

remove(0)は0番を消し、remove("0")は値としての"0"を探します。型を見ます。そこです。

Tracea Note

カードを押すと、ここに深掘り解説が表示されます。

まずは、戻り値を捨てていないか、本体がその場で変わるのかを分けて読みます。Stringは変わらない。StringBuilderとArrayListは変わる。この対比を置いてから、1行ずつ値を追いましょう。
