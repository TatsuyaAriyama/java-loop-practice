Silver Focus 04

例外処理は、catchより先に「どこで例外が起きるか」を読む。

try、catch、finallyは形だけなら簡単です。ただし試験では、例外が起きた行より後が実行されるのか、catchの順番は正しいのか、finallyが最後に走るのかを問います。

try {
  System.out.println("A");
  int n = 10 / 0;
  System.out.println("B");
} catch (ArithmeticException e) {
  System.out.println("C");
} finally {
  System.out.println("D");
}
try

tryは危ない処理を囲む

例外が起きると、その行から先のtry内処理は飛ばされ、対応するcatchを探します。

catch

catchは上から順に判定する

親クラスの例外を先にcatchすると、子クラスのcatchへ到達できません。順番でコンパイルエラーになります。

finally

finallyは原則最後に走る

例外が起きても起きなくても、finallyは実行されます。出力順の最後に入ることが多いので、忘れないでください。

checked

チェック例外は処理が必要

コンパイル時に処理を求められる例外があります。throwsかtry-catchが必要かを見る問題があります。

runtime

実行時例外は突然出る

ArithmeticException、NullPointerException、ArrayIndexOutOfBoundsExceptionなどは読解でよく出ます。

flow

出力は例外発生行で切れる

例外が起きた行より後のtry内printlnは出ません。どの文字が表示されないかを先に消します。

Tracea Note

例外問題は、出力順を声に出すくらい丁寧に。

上のコードなら、Aを表示したあとにゼロ除算で例外が起きます。Bには進みません。ArithmeticExceptionのcatchに入りCを表示し、最後にfinallyでDを表示します。結果はA、C、Dです。

もしcatchの順番が親から子になっていたら、実行以前にコンパイルエラーです。Silverでは「実行結果」だけでなく「そもそも通るコードか」を必ず先に見ます。
